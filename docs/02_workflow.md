# 核心逻辑流

## 1. 总流程

```text
用户上传资料 / 输入业务问题
→ 资料读取
→ 文档类型识别
→ 任务类型判断
→ 选择分析链路
→ 关键信息提取
→ 结构化摘要
→ 风险点识别
→ 商业价值判断
→ 执行方案生成
→ 待核验项输出
→ 复核清单输出
→ 生成报告 / 表格 / 行动清单
```

## 2. 长链推理设计

本项目包含长链推理。

原因是企业资料分析不是一次性摘要，而是连续判断链：

```text
先判断资料类型
→ 再提取事实
→ 再归纳结构
→ 再识别风险
→ 再判断商业价值
→ 再生成行动方案
→ 最后输出复核清单
```

每一步都依赖前一步结果。

## 3. 多 Agent 协作设计

当前 Agent 角色：

1. DocumentClassifierAgent  
   判断资料类型。

2. FactExtractionAgent  
   提取文档中的明确事实。

3. StructureSummaryAgent  
   生成结构化摘要。

4. RiskReviewAgent  
   识别风险和不确定项。

5. BusinessAnalysisAgent  
   判断商业价值、市场机会和业务切入点。

6. ActionPlanAgent  
   生成执行方案。

7. ChecklistAgent  
   生成复核清单。

8. ReportWriterAgent  
   生成最终报告。

## 4. 不同资料类型的分支链路

### 政策文件

```text
政策原文
→ 政策类型识别
→ 政策要点提取
→ 适用对象判断
→ 机会窗口识别
→ 落地建议
→ 待核验项
```

### 合同草案

```text
合同原文
→ 合同类型识别
→ 双方主体提取
→ 权责条款提取
→ 付款与违约条款识别
→ 风险条款审查
→ 律师沟通清单
```

### 竞品资料

```text
竞品资料
→ 公司/产品识别
→ 业务范围提取
→ 商业模式分析
→ 客户类型判断
→ 市场缺口分析
→ 可借鉴策略
```

### 项目方案

```text
项目方案
→ 项目目标识别
→ 资源需求提取
→ 执行路径梳理
→ 风险阻塞识别
→ 预算收益判断
→ 下一步行动清单
```

### 内部培训材料

```text
培训资料
→ 知识点识别
→ 学习路径设计
→ 操作步骤拆解
→ 常见误区提取
→ 考核标准生成
```
