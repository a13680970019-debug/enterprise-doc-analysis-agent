# 多 Agent 角色设计

## 1. DocumentClassifierAgent

职责：

- 判断文档类型
- 判断任务类型
- 决定后续调用哪条分析链路

输入：

- 文档标题
- 文档正文
- 用户问题

输出：

- document_type
- task_type
- suggested_chain

## 2. FactExtractionAgent

职责：

- 只提取文档中明确出现的信息
- 不做主观推测
- 标注事实来源位置

输出：

- 主体
- 时间
- 金额
- 条款
- 条件
- 关键定义
- 重要事件

## 3. StructureSummaryAgent

职责：

- 把事实转化为结构化摘要
- 按业务使用场景分类整理

输出：

- 摘要
- 表格
- 分类结构
- 关键判断

## 4. RiskReviewAgent

职责：

- 识别合同风险
- 识别政策适用风险
- 识别商业落地风险
- 识别信息不完整风险

输出：

- 风险点
- 风险等级
- 风险原因
- 建议处理方式

## 5. BusinessAnalysisAgent

职责：

- 判断商业价值
- 识别市场机会
- 分析竞品模式
- 判断业务切入点

输出：

- 商业机会
- 市场缺口
- 客户类型
- 可落地场景
- 收益判断

## 6. ActionPlanAgent

职责：

- 将分析结果转化为行动方案
- 明确下一步动作
- 明确执行优先级

输出：

- 行动清单
- 优先级
- 负责人建议
- 时间节奏
- 交付物

## 7. ChecklistAgent

职责：

- 生成复核清单
- 区分已确认信息和待核验信息
- 防止推测冒充事实

输出：

- 复核问题
- 待核验项
- 风险确认项
- 人工确认建议

## 8. ReportWriterAgent

职责：

- 生成最终报告
- 适配管理层、业务部门、律师、销售或培训场景

输出：

- Markdown 报告
- 表格
- 汇报框架
- 行动清单
