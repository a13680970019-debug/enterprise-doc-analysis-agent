# Enterprise Document Analysis & Decision Support Agent

企业资料分析与决策支持 Agent。

本项目用于帮助团队处理政策文件、行业报告、项目方案、合同草案、竞品资料和内部培训材料，将非结构化资料转化为结构化摘要、风险点清单、商业价值判断、执行方案、复核清单和可用于汇报的报告。

---

## 1. 项目解决的核心痛点分析

### 痛点 1：企业决策依赖大量非结构化文档

企业日常决策需要阅读政策文件、行业报告、项目方案、合同草案、竞品资料和内部培训资料。这些资料格式不统一、篇幅长、信息密度高，人工阅读成本高。

**系统解决方式：**

通过资料读取、文档类型识别、关键信息提取和结构化摘要，将原始资料转化为可阅读、可复核、可执行的信息结构。

---

### 痛点 2：人工提炼信息不稳定，容易遗漏关键风险

不同人员阅读同一份资料，提取出的重点可能不同。政策适用条件、合同风险条款、竞品业务边界、项目执行阻塞点等信息容易被遗漏。

**系统解决方式：**

系统设置多轮分析链路：

1. 第一轮：事实提取
2. 第二轮：结构化归纳
3. 第三轮：风险审查
4. 第四轮：业务报告生成
5. 第五轮：待核验项与复核清单输出

---

### 痛点 3：资料阅读结果难以快速转化为业务动作

很多企业阅读资料后只停留在“知道了”，无法快速转化为汇报材料、行动清单、律师沟通清单、市场缺口分析或项目推进计划。

**系统解决方式：**

Agent 不只做摘要，还会根据不同资料类型输出对应的业务化结果：

- 政策文件：政策要点、适用对象、机会窗口、落地建议
- 合同草案：双方权责、风险条款、律师沟通清单
- 竞品资料：业务范围、商业模式、客户类型、市场缺口
- 项目方案：目标拆解、资源要求、执行路径、风险阻塞
- 培训材料：学习路径、培训大纲、操作清单、考核标准

---

### 痛点 4：重要结论缺少依据来源，难以复核

人工整理资料时，经常出现结论和原文依据分离的问题，后续很难判断某个结论来自哪里。

**系统解决方式：**

对于重要结论，Agent 会要求标注依据来源。无法确认的信息单独列为“待核验项”，避免把推测当成事实。

---

### 痛点 5：企业知识无法沉淀成长期资产

大量分析工作重复发生，但结果分散在个人文档、聊天记录和临时报告里，无法沉淀成企业知识库和标准分析链路。

**系统解决方式：**

项目后续可接入企业知识库、多 Agent 协作流程、文件批量处理、报告自动生成和执行复盘系统，形成完整的企业决策支持底座。

## 2. 核心逻辑流

整体链路：

```text
用户上传资料 / 输入业务问题
→ 资料读取
→ 文档类型识别
→ 任务类型判断
→ 选择分析链路
→ 关键信息提取
→ 结构化摘要
→ 风险点识别
→ 商业价值判断
→ 执行方案生成
→ 待核验项输出
→ 复核清单输出
→ 生成报告 / 表格 / 行动清单
```

---

## 3. 是否包含长链推理

包含。

本项目不是一次性问答，而是多阶段连续分析：

```text
事实提取
→ 结构化归纳
→ 风险审查
→ 商业判断
→ 业务报告生成
→ 人工复核清单
```

前一阶段的输出会作为后一阶段的输入，形成可追踪的长链推理。

---

## 4. 是否包含多 Agent 协作

包含。项目设计为多 Agent 架构。

当前规划角色：

| Agent | 职责 |
|---|---|
| DocumentClassifierAgent | 判断文档类型 |
| FactExtractionAgent | 提取文档事实 |
| StructureSummaryAgent | 结构化摘要 |
| RiskReviewAgent | 风险审查 |
| BusinessAnalysisAgent | 商业价值判断 |
| ActionPlanAgent | 执行方案生成 |
| ChecklistAgent | 复核清单生成 |
| ReportWriterAgent | 报告生成 |

## 5. 典型应用场景

### 5.1 政策文件分析

输出：

- 政策要点
- 适用对象
- 申报条件
- 机会窗口
- 企业可落地建议
- 待核验事项

### 5.2 合同草案分析

输出：

- 合同双方主体
- 权利义务
- 付款条款
- 违约责任
- 风险条款
- 律师沟通清单

### 5.3 竞品资料分析

输出：

- 业务范围
- 商业模式
- 客户类型
- 产品能力
- 市场缺口
- 可借鉴策略

### 5.4 项目方案分析

输出：

- 项目目标
- 关键资源
- 执行路径
- 风险阻塞
- 预算与收益判断
- 下一步行动清单

### 5.5 内部培训材料生成

输出：

- 培训大纲
- 学习路径
- 操作步骤
- 常见误区
- 检查清单
- 考核标准

## 6. 项目运行方式

当前版本是基础工程骨架，可直接上传 GitHub，并可运行一个简单的命令行演示。

```bash
python src/main.py --type policy --input examples/policy_example.txt
```

也可以分析合同示例：

```bash
python src/main.py --type contract --input examples/contract_example.txt
```

## 8. 安全注意事项

不要上传以下内容到公开 GitHub 仓库：

- 客户资料
- 合同原文
- 内部政策文件
- 企业数据库
- API Key
- `.env` 文件
- 真实项目报价
- 真实客户线索
